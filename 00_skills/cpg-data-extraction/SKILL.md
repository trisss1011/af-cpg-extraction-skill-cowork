# CPG 메타분석용 데이터 추출 스킬

## 목적

심방세동 한의CPG 체계적문헌고찰에 필요한 정량 데이터를 추출한다.
산출물은 **논문 1건당 1개**의 단독 엑셀 파일(`90_Output/extracts/AF_extract_<번호>_<study_id>.xlsx`)이다. 이 파일들은 `merge-skill`이 마스터(`AF_CPG_data_extraction_심상송.xlsx`)로 일괄 통합한다. 근거표 작성·RevMan 매핑·RoB 판정은 후속 스킬에서 수행한다.
추출된 데이터는 연구자가 반드시 원문과 대조 확인한다.

### 왜 단독 파일로 저장하는가

동시 여러 세션이 마스터 엑셀에 직접 쓰면 openpyxl의 `load_workbook` → `save` 구조상 **lost update**(쓰기 손실) 경쟁 조건이 발생한다. 세션 B의 save가 세션 A의 변경분(다른 논문 데이터)을 소리 없이 덮어쓴다. 이 문제는 파일 락이나 재시도로는 근본 해결되지 않는다. 해결책은 추출과 병합의 분리: 추출은 세션별 단독 파일에, 병합은 단일 시점에 원자적으로 수행한다.

---

## 작업 흐름

```
PDF 읽기 → 기본정보 추출 → RoB 근거 수집 → 한약 중재 추출 → 아웃컴 목록화 → 데이터 추출 → 채팅 출력 → 연구자 확인 → 세션별 단독 파일 저장
```

### 후속 단계

1. **병합**: 논문별 추출 파일이 쌓이면 `merge-skill`이 마스터(`AF_CPG_data_extraction_심상송.xlsx`)로 통합한다.
2. **분석 파이프라인**: 통합된 마스터는 별도 스킬(RevMan Export, RoB 판정)의 입력으로 사용된다.

### 마스터 파일에 직접 쓰지 않는 이유

동시 추출 세션 중 한 세션이 마스터를 save하면 다른 세션의 미반영 변경분이 소실된다 (lost update). 본 스킬은 이를 방지하기 위해 **마스터 파일을 절대 수정하지 않는다**. 마스터 수정은 오직 `merge-skill`만 수행한다.

## 핵심 원칙

1. **원문 근거 절대 원칙**: 논문 원문에 명시된 내용만 기재한다. 문맥상 추측 가능하더라도 직접 기재되지 않은 정보는 절대 채워넣지 않는다.
2. **수치 원문 준수**: 수치는 원문 그대로 기록한다. 계산·변환 시 반드시 비고에 명시한다.
3. **미보고 처리**: 논문에 없는 정보는 **NR** (Not Reported)로 기록한다.
4. **표 우선 원칙**: 표(Table)와 본문(Text) 수치가 다를 경우 **표 수치 우선** 채택, 비고에 불일치 기록.
5. **단위 표기**: 시간 단위는 영문약어로 통일한다 — `d`(일), `wk`(주), `mo`(개월), `yr`(년). 기타 단위는 원문 준수 (min/h/beats/min/mm/% 등).
6. **군 방향 혼동 금지**: E군(치료군)과 C군(대조군) 방향을 혼동하지 않는다. E/C 병기 시 항상 E 먼저.
7. **SD vs SE 구분**: 불명확하면 비고에 "SD/SE 불명확" 기록. 임의 판단 금지.
8. **자료 유형 판별**: 연속형/이분형을 정확히 판별한다.
9. **측정 시점 구분**: Baseline/After/Change를 반드시 구분한다.
10. **번호 확인**: PDF 파일명 맨 앞 숫자가 논문 번호다 (예: `30_2023_LI_...pdf` → 번호=30). 선정/배제 관리 파일을 별도로 조회할 필요 없다.
11. **한약 중재**: 별도 세부 필드로 추출한다 — `references/korean-medicine-intervention.md` 참조.
12. **1건씩 처리**: 한 논문씩 처리하고 연구자 확인 후 다음으로 넘어간다.
13. **p값 표기**: 소문자 `p` 통일 (예: `p>0.05`, `p<0.01`). 대문자 `P` 사용 금지.

## 1단계: PDF 읽기 및 연구 설계 파악

PDF 전체를 읽고 다음을 파악한다.

**연구 설계 분류**:
- RCT: 무작위배정 대조군 임상시험
- quasi-RCT: 무작위배정 불충분/불명확 ("随机分为"만 있으면 해당)
- non-RCT: 비무작위 대조연구(CCT 포함)
- 관찰연구: 코호트, 환자-대조군, 단일군 전후

**PICO 파악**:
- P: 환자 특성 (진단명, AF 유형, 유병기간, 동반질환)
- I: 한약 중재
- C: 비교군 (위약/표준치료/양방치료/다른 한약)
- O: 결과지표 목록 → `references/af-outcomes.md` 참조

**중국어 논문 주의사항**:
- 표 수치와 본문 수치 불일치 시 표 우선
- 总有效率(총유효율), 治愈率(치유율), 复发率(재발률) 등 약어 확인

## 2단계: 기본정보 추출

추출 결과 파일의 열 구성은 다음과 같다.

### 2A. 식별 및 관리 정보

| 열 | 항목 | 설명 |
|----|------|------|
| A | **번호** | 선정/배제 관리 파일(시트1) 번호와 동일 |
| B | study_id | 저자_연도 (예: Zhang_2019) |
| C | author | First author et al. |
| D | year | 출판연도 |
| E | title | 논문 제목 |
| F | journal | 저널명 |
| G | vol_issue | 권(호) |
| H | pages | 페이지 |
| I | doi_pmid | DOI 또는 PMID |
| J | **exclude** | Y / 공백 — 연구자가 검토 후 기재 |
| K | **exclude_reason** | 배제 이유 — 연구자 기재 (PRISMA flow 작성용) |

### 2B. 연구 특성

| 열 | 항목 | 설명 |
|----|------|------|
| L | country | 연구 수행 **국가명만** (도시·지역 제외). 예: `中国` 또는 `China` |
| M | setting | **단일/다기관 + 입원/외래만** 기재 (기관명·연구기간 제외). 예: `단일기관, 입원` / `다기관, 외래`. 미명시는 `NR` |
| N | study_design | RCT / quasi-RCT / non-RCT / 관찰연구 |
| O | sample_size | 초기 E:C → 최종 분석 E:C |
| P | age_E / age_C | 평균±SD 또는 Median(IQR). **E 먼저 기재**: `E: 값 / C: 값` |
| Q | sex_E / sex_C | 형식 고정: `E: <남>M/<여>F / C: <남>M/<여>F`. 예: `E: 32M/28F / C: 34M/26F` |

### 2C. AF 유형 및 동반질환

| 열 | 항목 | 코드 체계 |
|----|------|---------|
| R | **af_type_code** | **코드 숫자만 기재** (라벨·텍스트 금지). 1=발작성, 2=지속성, 3=영구성, 4=혼합/미특정 |
| S | **af_type_other** | **코드 숫자만 기재**. 5=AF with RVR, 6=NVAF(논문 명시 시만), 7=기타. 해당 없으면 공란. |
| T | af_type_text | AF 유형 원문 텍스트 |
| U | **comorbidity_code** | **코드 숫자만**. P 동반질환 명시 시만. 일반 AF=`NA`. 1=RFCA(중복허용), 2=심부전, 3=심근경색/관상동맥질환, 4=갑상선/기타, 5=고혈압. 복수=쉼표. |
| V | comorbidity_text | 동반질환 조건 원문. NA이면 `NA` |
| W | tcm_pattern | 중의 변증(辨證) 포함 조건 명시 시 기재. 없으면 `NR` |
| X | disease_duration | 이환기간 — 수치는 원문 그대로, **시간 단위는 영문약어** 통일 (`d`/`wk`/`mo`/`yr`). 예: `E: 2.27±0.98yr / C: 2.98±0.65yr` |
| Y | baseline_comparable | 기저치 동질성 (Yes/p= 또는 NR). **소문자 `p`** 사용. 예: `Yes (p>0.05)` |

### 2D. 중재 및 비교군

| 열 | 항목 | 설명 |
|----|------|------|
| Z | **comparison_type** | 비교 유형 코드 (아래 표 참조) |
| AA | intervention_E | 한약 처방명. 형식: `漢字 (romanization)` |
| AB | intervention_C | 대조군 중재 이름만 |
| AC | intervention_C_method | 대조군 복용/시행 방법. 형식: `투여경로, 용량×횟수/일` |
| AD | co_intervention | 차별 부가중재만. 아래 규칙 참조. |
| AE | treatment_period | 치료 기간 — 시간 단위 영문약어 (`4wk`, `3mo` 등) |
| AF | treatment_sessions | 총 치료 횟수 |
| AG | follow_up | 추적 관찰 기간 — **치료 종료 후 별도로 관찰한 기간**. 없으면 `NR`. 영문약어 (`1yr`, `6mo` 등). |
| AH | funding | 연구비 출처 |
| AI | outcomes_reported | 논문에서 **보고된 모든** 아웃컴 목록 (표준 외 항목 포함, **세미콜론** 구분). |

**comparison_type 코드 체계**:

| 코드 | 의미 |
|------|------|
| `KM_alone_vs_placebo` | 한방 단독 vs 위약/무처치 |
| `KM_alone_vs_WM` | 한방 단독 vs 양방 단독 |
| `KM+WM_vs_WM` | 한방+양방 병행 vs 양방 단독 |
| `KM+WM_vs_KM` | 한방+양방 병행 vs 한방 단독 |

**co_intervention (AD열) 기재 규칙**: 차별 부가중재만 서술형으로 기재. 공통 배경치료는 제외.

| 상황 | 기재 예시 |
|------|---------|
| 차별 부가중재 없음 (또는 공통만 존재) | `NR` |
| 치료군만 부가 | `아미오다론 0.2g×1회/일 (치료군)` |
| 대조군만 부가 | `생활습관 교육 (대조군)` |
| 양쪽 모두 차별 | `아미오다론 (치료군); 생활습관 교육 (대조군)` |

복수 항목은 세미콜론(`;`)으로 구분.

> **한약 중재의 복용방법**: 기본정보 시트에 기재하지 않는다. `한의중재_한약` 시트의 administration열(G열)에서 관리.

**intervention_E/C/method (AA~AC) 기재 규칙**:

| 열 | 형식 | 예시 |
|----|------|------|
| AA intervention_E | `漢字 (romanization)` — **이름만**. 제형·용량·빈도 제외 | `参松养心 (Shensong Yangxin)` |
| AB intervention_C | 대조군 중재 **이름만** | `Dronedarone` |
| AC intervention_C_method | `투여경로, 용량×횟수/일` | `경구, 400mg×2회/일`. 복잡 용법(주별 감량 등)은 서술형: `경구, 0.2g×3회→2회→1회/일 (주별 감량, 총 4wk)` |

**outcomes_reported (AI열) vs 아웃컴 시트**: AI열은 논문의 **모든** 아웃컴을 한 셀에 세미콜론으로 나열(전체 조망용). 아웃컴 시트는 **모든** 아웃컴을 행별로 추출하며 표준·비표준 구분은 F열(importance)로 한다.

RoB 2.0 판정용 **11열 모두 추출** (누락 금지). 판정은 하지 않고 원문/사실/수치만. 미보고=`NR`. Methods·Results 1회 읽기로 병행.

|----|------|---------|-----------------|

| AK | rob_d1_concealment | 원문 | 배정 순서 은폐 수단 원문 |
| AL | rob_d1_baseline | 사실 | 두 군 기저치 비교에서 p<0.05 변수 존재 여부·변수명 |
| AM | rob_d2_blinding | 원문 | 환자·시술자 눈가림 방식, 위약/샴 사용 원문 |
| AN | rob_d2_analysis | 사실 | 통계 분석 집단 기준 (`ITT`/`PP`/`mITT`/NR) |
| AO | rob_d3_dropouts | 수치 | 군별 초기 배정 인원·최종 분석 인원 (또는 탈락자 수) |
| AP | rob_d3_reasons | 원문 | 탈락·결측치 사유 원문 |
| AQ | rob_d4_assessor | 원문 | 결과 평가자·통계 분석가 눈가림 원문 |
| AR | rob_d5_protocol | 사실 | 임상시험 사전 등록 번호 (예: ChiCTR…) |
| AS | rob_d5_outcome | 사실 | Methods 계획 아웃컴이 Results에 누락 없이 보고되었는지 |
| AT | rob_other_coi | 원문 | 연구비 지원·이해상충(COI) 선언 문구 |

도메인 분류·예시·기존 열과의 관계 등 상세는 `references/rob-extraction-fields.md` 참조.

### 2F. 작업자 비고 (v2.4 신규)

| 열 | 항목 | 기재 유형 | 기재 내용 |
|----|------|---------|-----------|
| AU | **notes** | 사용자 | 추출 후 작업자가 해당 논문의 특이사항을 자유 기록하는 열 |

**추출 단계 기재 규칙**

- **Claude는 추출 시 이 열을 공란으로 남긴다** (NR도 쓰지 않음).
- 논문 특이사항(예: 원문 수치 의심, 중복 게재 의심, RoB 판정 재검토 필요, 데이터 재확인 요청 등)은 작업자가 추출 후 자유 기술한다.
- 본 열은 메타분석 투입 데이터가 아니라 작업 관리용 메모이다.
- 머지 시 이 열의 내용도 그대로 마스터에 반영된다 (작업자의 메모가 보존됨).

> **혼동 주의**: 아웃컴 시트에도 동일 이름의 `notes`열(U열)이 있으나 용도가 다르다. 아웃컴의 notes는 데이터 변환 내역·세부 이상반응 등 추출 근거 기록용이고, 본 기본정보의 notes는 작업자 전반 메모용이다.

## 3단계: 한약 중재 추출

한약 중재 추출 상세 기준은 `references/korean-medicine-intervention.md` 참조.

### 3A. 한약 처방 구성 — `한의중재_한약` 시트

한약 중재 논문은 **별도 시트에 처방 구성 및 복용방법을 원문 그대로 추출**한다. 1논문 = 1행 (wide form). 주처방만 기록, 부처방은 note에 서술.

| 열 | 필드명 | 설명 |
|---|---|---|
| A | 번호 | 기본정보 시트와 동일 |
| B | study_id | 기본정보 시트와 동일 |
| C | formula_name_kr | 처방명 (한글) |
| D | formula_name_cn | 처방명 (한자) |
| E | formulation | 탕약 / 과립 / 캡슐 / 환제 / 주사제 / NR |
| F | composition | 약재+용량 원문 (한 셀, 쉼표 구분, 원문 순서 보존) |
| **G** | **administration** | 투여경로·용량·횟수. 예: `경구, 탕약 수전 400mL, 200mL×2회/일` |
| H | note | 비고 (부처방, NR 사유 등) |

**F열(composition) 기재 규칙**:
1. 형식: `약재명` + 공백 + `용량원문단위` (예: `麻黃 9g`)
2. 구분자: 쉼표(`,`) + 공백으로 통일. 원문의 `、` `·` `；`은 쉼표로 변환.
3. **번역·단위환산 금지** — 약재명·용량 모두 원문 언어·단위 그대로.
4. **법제 표기 유지** — `生·炒·炙·蜜炙·制·焦·煨` 등 포제 접두어는 **원문 표기 그대로** 기재 (예: `生黄芪 25g`, `炙甘草 8g`, `炒枣仁 15g`). 임의 제거·정규화 금지.
5. 한국 논문이 한자 병기면 D·F에 한자 우선.
6. `各9g` 같은 공통 표기는 풀어서 개별 기록, note에 `"원문 '各9g' 풀어 기재"` 명시.
7. **약재 전체 용량 미기재** (중성약·시판 처방에서 약재명만 나열된 경우): F열에 약재명만 쉼표 구분으로 나열 (예: `麻黃, 桂枝, 杏仁, 甘草`). H열(note)에 `용량 미기재` 명시.
8. **처방 전체 구성 비공개** (성분 비공개 캡슐 등): F열 `NR`, H열에 사유.

**G열(administration) 기재 규칙**:
- 형식: `투여경로, 제형 용량, 1회량×횟수/일 (복용법)`
- 예: `경구, 탕약 수전 400mL, 200mL×2회/일 (분조석 온복)`
- 예: `경구, 캡슐, 1.6g×3회/일`
- 정보 미보고 시 `NR`.

**다중 처방 처리**: 주처방만 행 1개. 부처방은 note에 서술 (예: `부처방: 생맥산(心氣虛 환자 대상)`).

**대조군 한약 처리**: 이 시트는 **치료군(E) 한약 중재만** 대상. 대조군 한약은 제외 (KM vs KM 비교 연구는 별도 확장 시 고려).

## 4단계: 아웃컴 목록화 및 자료 유형 판별

논문에 보고된 **모든 아웃컴**을 대상으로 한다 (표준 + 비표준). 표준 아웃컴 목록은 `references/af-outcomes.md` 참조 — 이 목록에 있으면 Critical/Important 등급을 부여하고, 없으면 F열(importance)을 공란으로 둔다.

**outcome_std (C열) 기재 규칙**:
- **표준 아웃컴**: `af-outcomes.md` §3 매핑표의 C열 코드를 사용. 약어 우선, 없으면 짧은 영문명.
- **비표준 아웃컴**: 통용 약어 있으면 약어 (예: `LVEDD`, `BNP`), 없으면 짧은 영문명 (예: `TCM symptom score`).
- D열(outcome_original)에 원문 용어가 기록되므로 C열은 간결한 식별자 역할.

**자료 유형 판별 기준**:

| 자료 유형 | 판별 기준 | 추출 형식 |
|---------|---------|---------|
| 연속형 | Mean±SD, Mean(95%CI), Median(IQR) | Mean, SD, N |
| 이분형 | n/N, %, event수/총수 | Event, Total |
| 순서형 | 등급별 인원수 (현효/유효/무효) | 이분형 변환 후 추출 |
| 효과크기 직접 보고 | OR, RR, HR, MD, SMD + CI | 수치 그대로 추출 |
| 사건발생시간 | Kaplan-Meier, HR | HR, 95%CI 또는 log(HR), SE |

**순서형 → 이분형 변환 규칙** (총유효율 패턴):
- (현효 + 유효) = Event, 총수 = Total
- 비고란에 "순서형→이분형 변환: 현효+유효=Event" 기록

### 아웃컴 시트 열 구성

| 열 | 항목 | 설명 |
|----|------|------|
| A | 번호 | 기본정보 시트와 동일 |

| H | E_n | 치료군 N |
| I | E_val1 (Mean/Event) | 치료군 평균 또는 사건수 |
| J | E_val2 (SD/Total) | 치료군 SD 또는 총수 |
| K | C_n | 대조군 N |

| N | effect_type | OR / RR / MD / SMD / HR |
| O | effect_value | 효과크기 값 |
| P | ci_lower | 95%CI 하한 |

| S | direction | 치료군 유리 / 대조군 유리 / 차이없음 |

#### time_point (G열) 기재 규칙

측정 시점을 시간값으로 기재하되, **치료 종료 후 추적 관찰(follow-up) 시점에만 `FU_` 접두사**를 붙인다. 치료기간 내 측정(Baseline 포함)은 시간값만 기재.

| 상황 | 기재 형식 | 예시 |
|------|---------|------|
| 치료 전 기저치 | `Baseline` | `Baseline` (**Before 사용 금지**) |
| 치료 중간 측정 | 시간값 | `2wk` (4주 치료 중 2주차 측정) |
| 치료 종료 시점 | 시간값 | `4wk` (4주 치료 종료 시 측정) |
| 치료 종료 후 추적 관찰 | `FU_` + 시간값 | `FU_1yr` (치료 종료 후 1년 시점) |
| 이상반응 (치료기간 전체) | `치료기간 중(기간)` | `치료기간 중(4wk)` |

**판별 기준**: 해당 측정이 AE열(치료기간) 이후에 별도로 수행된 관찰이면 `FU_` 접두사. AG열(follow_up)에 기간이 명시된 논문에서 해당 시점의 데이터가 있으면 `FU_`에 해당.

**시간값 표기**: 영문약어는 핵심원칙 5 참조. 예: 3개월→`3mo`, 14일→`14d` 또는 `2wk`.

#### direction (S열) 기재 규칙

**기본값**: `치료군 유리` / `대조군 유리` / `차이없음` (3가지)

**비직관적 방향일 때만 괄호 부연 허용**: 수치 방향과 임상적 해석이 다른 경우.
- 예: `치료군 유리 (HR 높음)` — HR 높음이 일반적으로는 불리이지만, dronedarone 서맥 부작용 완화 맥락에서 유리

원시데이터 또는 효과크기가 있으면 빈칸 허용 (방향은 수치에서 판단 가능). 정량 데이터 없이 서술형으로만 결과가 보고된 경우 반드시 기재.

#### event_direction (T열) 허용값

| 값 | 용도 |
|----|------|
| `Event=유효` | 총유효율, 동율동전환율 등 긍정적 결과가 Event |
| `Event=재발(불량결과)` | AF 재발률 등 부정적 결과가 Event |
| `Event=이상반응(불량결과)` | 이상반응 발생 |
| `N/A` | 연속형 아웃컴 |

위 4가지 외의 값 사용 금지.

#### p_value (R열) 기재 규칙

| 상황 | 기재 형식 | 예시 |
|------|---------|------|
| 정확한 수치 보고 | 숫자 | `0.03` |
| 부등호만 보고 | 부등호 문자열 | `<0.01`, `<0.05`, `>0.05` |
| 미보고 | `NR` | `NR` |

#### V열 (val2_type) 기재 규칙

- **연속형 행**: 기본값 `SD`. 논문이 `Mean±SE`로 보고한 경우만 `SE`.
- **이분형 행**: **공란** (val2가 Total이므로 SD/SE 개념 부적용).
- SD/SE 불명확 시: `SD`로 기재하되 notes(U열)에 `"SD/SE 불명확"` 기록.
- RevMan Export 시: val2_type=SE이면 SD 변환 없이 SE 열에 직접 매핑.

> **data_class(Arm/Contrast), effect_measure(LOG_OR 등) 열은 v2.2부터 제거**. 후속 RevMan Export 스킬이 H~M·N~Q 채움 여부와 effect_type(N열)에서 자동 산출한다.

### 5.1 연속형 데이터

측정 시점이 여러 개인 경우 각 시점을 별도 행으로 추출.

| | Before | After | Change |
|---|---|---|---|
| **치료군(E)** | M_E.before, SD_E.before, N_E | M_E.after, SD_E.after, N_E | M_E.change, SD_E.change, N_E |
| **대조군(C)** | M_C.before, SD_C.before, N_C | M_C.after, SD_C.after, N_C | M_C.change, SD_C.change, N_C |

**SD 변환** (필요시 비고 명시): SE→SD=`SE×√N` / 95%CI→SD=`√N×(U−L)/3.92` / IQR→SD≈`IQR/1.35` (정규분포 가정)

### 5.2 이분형 데이터

| | Event | Total |
|---|---|---|
| **치료군(E)** | E_event | E_total |
| **대조군(C)** | C_event | C_total |

- %만 보고된 경우: N×% 역산, 비고에 기록
- Event 방향 필수 기록: T열에 `Event=유효` 또는 `Event=재발(불량결과)` 등

### 5.3 이상반응 (Adverse Events) — 추출 방식

이상반응은 **아웃컴 시트에 `AE total` 1행만 추출**한다 (논문마다 세부 유형이 다르므로 세부 유형별 별도 행 분리 불필요).

#### 이분형 행 구성 (아웃컴 시트)

| outcome_std | data_type | Event_E | Total_E | Event_C | Total_C | notes |
|---|---|---|---|---|---|---|
| AE total | 이분형 | n | N | n | N | 두통 X건(E m/C m), 두현 X건(E m/C m), 위장반응 X건(E m/C m) |

**추출 규칙**:
- `AE total` 행 1개만 추출. Event = 이상반응 발생 건수 합계, Total = 각 군 전체 N.
- 논문에서 이상반응 전체 합산값을 직접 보고한 경우 그 값을 기재한다.
- 세부 유형별 합산이 전체와 다를 경우 notes에 명시한다.

**고찰용 서술 (`notes` 열)**:
- 세부 유형명과 각 건수를 텍스트로 기록한다.
- 형식: `"두통 X건(E n/C n), 두현 X건(E n/C n), 위장반응 X건(E n/C n)"`
- 건수 미보고 시: `"보고 유형: 두통, 두현, 위장반응 (세부 건수 NR — 원문 수기 확인 필요)"`

### 5.4 효과크기 관련 열 (N~T) 추출 원칙

**핵심**: 메타분석 투입 데이터의 우선순위는 ① 원시데이터(H~M) > ② 효과크기+CI(N~Q) > ③ 검정통계량·p값. 원시데이터가 완전하면 소프트웨어(RevMan, R)가 효과크기·CI를 자동 산출하므로 N~Q는 빈칸 허용.

**상황별**:
- 원시데이터(H~M) 완전 → N~Q 빈칸, R은 원문 명시 시만 기록
- 원시 없음/불완전 → 보고된 효과크기·CI·p값을 **반드시 추출** (원시가 불완전하면 소프트웨어 재계산 불가)

**effect_type(N열) 주의**: OR/RR/HR/MD/SMD 등 **효과크기만** 기재. 검정통계량(t, χ², F, Z)은 N열 금지 — notes(U열)에 보조 기록.

**direction (S열)·event_direction (T열)**: 위 규칙 섹션 참조.

## 6단계: 연구자 확인 프로세스

### 6A. 처리 순서

```
PDF 추출 → 채팅창 출력 → 연구자 확인/수정 → 엑셀 저장
```

- **엑셀 저장은 연구자 확인 후에만 수행한다.** 확인 전에 엑셀에 먼저 저장하지 않는다.
- 연구자가 수정사항을 전달하면, 수정 내용을 반영한 뒤 엑셀에 추가한다.

### 6B. 채팅창 출력 내용

연구자에게 다음을 제시한다:

1. **추출 요약** — 아웃컴별 핵심 수치 마크다운 표
2. **불확실 항목** — NR 처리 이유, SD/SE 불명확 등
3. **변환 적용 내역** — SD 변환, 순서형→이분형 등
4. **한약 처방 확인 요청** — 처방 구성·법제·용량 등 전문가 검토 필요 항목
5. **comparison_type 확인** — 판별 불명확한 경우 표시
6. **RoB 11열 (AJ~AT) 추출 점검** — 11열 모두 값/NR 채워졌는지 확인. 누락 발견 시 보강 후 다음 진행.

확인·수정 후 엑셀 업데이트. 완료 후 다음 논문으로 진행.

### 6C. 채팅 출력 형식

**처리 방식: 단건 처리 원칙**
- 논문 1건씩 추출 → 채팅 출력 → 연구자 확인/수정 → 엑셀 저장 → 다음 논문
- 여러 논문을 동시에 처리하지 않는다 (데이터 혼입 방지)

**출력 구조**: 논문 원문과 대조하기 쉽도록 항목별로 구분하여 출력한다.

```
① 논문 서지정보      ← 표지/초록과 대조
② 연구 특성          ← 방법(Methods) 섹션과 대조
③ 중재 정보          ← 방법 > 치료방법 섹션과 대조
④ 아웃컴 데이터      ← 결과(Results) 표/본문과 대조
⑤ 불확실·NR 항목    ← 연구자 판단 요청
```

**세부 출력 규칙**:
- 각 항목을 `항목명: 값` 형식으로 **1항목 1줄** 출력
- 수치 데이터는 원문 표 구조를 그대로 재현한 마크다운 표로 출력
- ⚠️ 불확실/OCR불량/NR 항목은 값 뒤에 `⚠️` 표시
- 전문가 확인이 필요한 처방 구성은 별도 블록으로 분리
- 출력 마지막에 불확실 항목만 모아서 별도 번호 목록으로 정리

> 추출 데이터는 연구 보조 목적의 초안이다. 반드시 원문과 대조 확인할 것.

## 7단계: 출력 형식 (엑셀 저장)

### 7A. 세션별 단독 추출 파일

- **파일명**: `AF_extract_<번호>_<study_id>.xlsx`
  - `<번호>`: PDF 파일명 맨 앞 숫자 (예: `30`)
  - `<study_id>`: 기본정보 시트의 study_id 값 (예: `Li_2025`)
  - 결합 예시: `AF_extract_30_Li_2025.xlsx`
- **저장 위치**: `90_Output/extracts/` (폴더 없으면 자동 생성)
- **시트 구성**: 기본정보(47열) + 아웃컴(22열) + **한의중재_한약**(8열) — 3개 시트, `번호`로 연결
- **샘플 참조**: `sample_v2.4.xlsx`의 구조·서식을 동일하게 복제 (시트명·헤더·열 너비·서식·freeze panes 전부 일치)
- **마스터 파일 직접 수정 금지**: 추출 스킬은 `AF_CPG_data_extraction_심상송.xlsx`를 읽기 전용으로만 참조한다 (서식 복제 목적). 쓰기는 절대 금지.

### 7C. 저장 시점 주의사항

- 기존 동일 파일명이 있으면 덮어쓰기 전 사용자에게 확인 (재추출일 수 있음)
- 저장 후 파일 무결성 verify (행 수·헤더·핵심 값 재확인)
- 저장 완료 후 `merge-skill`로 마스터 통합이 필요함을 사용자에게 안내
