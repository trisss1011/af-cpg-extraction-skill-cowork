# v2.6 변경 명세 (cpg-data-extraction)

> 본 파일은 upgrade-skill이 마스터를 v2.5.x → v2.6으로 업그레이드할 때 적용하는 변경 항목을 정의한다.
> 버전 식별: `_meta` 시트의 `spec_version` 값이 `2.6` 미만이면 본 파일의 변경을 적용.
> 적용 후 `_meta`에 `spec_version=2.6`, `last_upgrade=<YYYY-MM-DD>` 기록.

---

## 변경 카테고리

자동 처리는 7단계 사용자 승인("예") 후 마스터에 즉시 반영. 사람 확인 항목은 7단계에서 변경하지 않고, **8단계 인터랙티브 처치**로 넘어가 카테고리별로 채팅창에 표시 → 작업자 응답대로 마스터에 즉시 반영. 병행해서 `.md` 리포트도 누적 저장.

| 카테고리 | 자동 처리 |
|---|---|
| **구조 마이그레이션**: 47열 → 48열 (AU `analysis_set` 신열, 기존 notes → AV) | ✅ |
| **AU `analysis_set` 코드화**: AN(rob_d2_analysis) → AU 코드(ITT/PP/NR) | ✅ |
| **HRV 행 삭제**: 아웃컴 시트에서 HRV 패밀리 행 제거 | ✅ |
| **AI열 HRV 토큰 제거**: 기본정보 outcomes_reported에서 HRV 토큰 제거 | ✅ |
| **study_design 재분류**: quasi-RCT → RCT/non-RCT 후보 | ❌ → 8단계 인터랙티브 |
| **study_design 비표준 값**: `RCT (prospective)` 등 자유서술 후보 | ❌ → 8단계 인터랙티브 |
| **af_type_other 코드 5 재검토**: HR≥110 명시 여부 | ❌ → 8단계 인터랙티브 |
| **SAE 추가 후보**: SAE 구성 사건 2개 이상 카테고리 매치 | ❌ → 8단계 인터랙티브 |

## 자동 처리 룰 상세

### 1. 구조 마이그레이션 (47→48열)

- 기본정보 시트 47열이고 헤더 끝이 `notes`이면 마이그레이션
- 기존 AU(`notes`) → AV로 시프트 (값·스타일 그대로)
- 신규 AU 헤더 `analysis_set`, 데이터행 공란
- 모든 셀 서식(폰트·정렬·배경·테두리·number_format) 보존
- 이미 48열이고 AU=`analysis_set`이면 skip (멱등)

### 2. AU `analysis_set` 코드화

기본정보 AN(`rob_d2_analysis`) 텍스트에서 분석집단 코드를 추출하여 AU에 채운다.

**키워드 매핑**:

| AN 토큰 | 매핑 |
|---|---|
| ITT, intention-to-treat, 의도-치료군, 의도치료군 | ITT |
| mITT, modified ITT, FAS, full analysis set | ITT (통합) |
| PP, PPS, per-protocol, per protocol, 프로토콜순응군 | PP |
| 위 명시 없이 `탈락자 제외`, `결측 제외`, `dropouts excluded` 등 PP 묵시 | PP |

**분류 알고리즘**:
1. NR로 시작하면 (`^\s*NR\b`) 즉시 `NR` 확정
2. ITT·PP 매치 직후 **15자 윈도** 안에 부정 키워드 있으면 매치 무효
   - 부정 키워드 화이트리스트(한국어): `명시 없음`, `명시 안 함`, `언급 없음`, `기술 없음`, `기재 없음`, `미명시`, `미 명시`
   - 부정 키워드 화이트리스트(영어): `not specified`, `not reported`, `not described`, `not stated`, `unclear`, `undescribed`, `unspecified`
   - ⚠️ 단독 `없음`/`안 됨`은 부정 키워드에서 **제외** (오탐 방지)
3. 살아남은 ITT·PP 매치가 둘 다 있으면 **첫 출현 우선**
4. ITT/PP 매치 없고 PP 묵시 서술 있으면 `PP`
5. 그 외 `NR`

**기존 AU 값 보존**: ITT/PP/NR 중 하나일 때만 보존. 자유서술/잔재는 덮어쓰기.

### 3. HRV 행 삭제

**HRV 패밀리 키워드** (엄격 매치):

| 카테고리 | 키워드 |
|---|---|
| 시간영역 | SDNN, SDANN, RMSSD, pNN50, NN50, HRV triangular index, 심박변이, 심율변이, 心率变异(性) |
| 주파수영역 | VLF, LF/HF, LF/HF ratio, TP, total power (LF/HF 단독은 오탐 차단 필터 거침) |
| 비선형·파생 | DC, AC, Poincaré, SD1, SD2, DFA, sample entropy, 심박감속능 |

**매치 알고리즘**:
1. `outcome_std` 셀이 키워드와 **정확히 일치** (대소문자 무시) → 삭제 대상
2. `outcome_std`/`outcome_original` 합산에서 **단어 경계 매치**
3. LF/HF 단독 매치 시 같은 셀에 `LVEF`/`HFmrEF`/`HFpEF`/`heart failure`/`left ventric`/`right ventric`/`atrial fibril`/`stroke`/`recurrence` 등이 있으면 매치 무효 (오탐 차단)

**삭제 처리**:
- 정확 일치는 즉시 삭제
- 단어경계 매치인데 효과크기·원시데이터(E_n/E_val1/effect_value)가 채워져 있으면 자동 삭제 보류 → ⚠️ 8단계 인터랙티브 처치 대상으로 표시 (작업자가 채팅에서 행별 응답)
- 삭제는 `ws.delete_rows`로 행 인덱스 역순 처리

### 4. AI열 outcomes_reported 정리

기본정보 AI열에서 HRV 토큰을 분리·제거:
- 구분자(`;`, `,`)로 분리
- 각 토큰에 HRV 키워드(단어 경계 매치) 포함되면 토큰 자체 제거
- 결과가 빈 문자열이면 `NR`

## 사람 확인 항목 검출 룰 (자동 변경 X)

### A. study_design 비표준 값

v2.6 표준값 {`RCT`, `quasi-RCT`, `non-RCT`, `관찰연구`} 외의 값은 후보로 표시.

정규화 추정:
- `Multi-centre RCT (...)`, `RCT (prospective)`, `RCT (1:1 무작위, ...)` → `RCT`
- `Systematic review / Meta-analysis` → exclude=Y 검토 (메타분석 대상 외)

### B. quasi-RCT 재분류 후보

현재 `study_design == quasi-RCT`인 행에 대해 AJ(`rob_d1_sequence`) + AK(`rob_d1_concealment`) 텍스트 분석:

- **무작위 표현 키워드**: `随机`, `随机分为`, `随机分组`, `随机法`, `以随机法`, `randomized`, `randomization`, `randomly divided/assigned/allocated`, `랜덤`, `무작위`
- **명시적 비무작위 키워드**: `入院顺序`, `入院时间`, `入院先后`, `门诊号`, `病案号`, `就诊时间`, `就诊顺序`, `visit order`, `admission order`, `alternation`, `alternating`, `by day`, `odd-even`, `格日`, `입원 순서`, `병상 번호`

분류 우선순위:
1. 비무작위 키워드 있음 → `non-RCT` 후보 (이 경우 무작위 표현 함께 있어도 non-RCT 우선)
2. 무작위 표현만 있음, 비무작위 없음 → `RCT` 후보
3. 무작위 표현 자체 없음 → `non-RCT` 후보

### C. af_type_other 코드 5 (AF with RVR) 검토

`af_type_other == 5`인 행에 대해 T(`af_type_text`)에서 임계값 110 명시 확인:

- 매치 패턴: `≥\s*110`, `>\s*=?\s*110`, `110\s*次/分`, `110\s*次/min`, `110\s*回/min`, `110\s*bpm`, `HR\s*≥\s*110`
- 매치 안 되면 ⚠️ "코드 5 해제 검토" 표시 (예: 100次/min만 명시되어 있는 케이스)

### D. SAE 추가 후보

같은 study_id 아웃컴 시트에 **SAE 구성 사건이 2개 이상 카테고리로** 추출된 경우만 후보.

**SAE 구성 사건 카테고리 + 키워드**:

| 카테고리 | 키워드 |
|---|---|
| 사망 | mortality, cardiac death, death, 사망률, 사망, 死亡 |
| 뇌졸중 | ischemic stroke, stroke, 허혈성 뇌졸중, 뇌졸중, TIA, 脑卒中, 脑梗 |
| 심부전 악화 | HF worsening/exacerbation, heart failure worsening/exacerbation, 심부전 악화, 心衰恶化, 心力衰竭加重 |
| AF 진행·재발 | AF recurrence/progression, persistent AF, permanent AF, AF 재발, 永久性 AF, 房颤复发, 房颤进展 |
| 혈전색전증 | thromboembolism, systemic embolism, 혈전색전, 全身性栓塞, 肺栓塞, 血栓栓塞 |
| 입원 | hospitalization/hospitalisation, 재입원, 住院 |
| 주요 출혈 | major bleeding, 대출혈, 主要出血, 大出血 |

**검출 알고리즘**:
1. 각 study_id별로 outcome_std/outcome_original에서 카테고리 매치 카운트
2. **2개 이상 카테고리** 매치 시 후보
3. 이미 SAE 행이 있는 study_id는 후보 제외
4. 8단계 채팅 표시 + `.md` 리포트에 각 후보의 매치된 사건 + 행번호 표시

**AF 재발 처리**: AF 재발은 efficacy outcome으로도 SAE 구성으로도 다뤄질 수 있어 본 스킬은 자동 판단하지 않는다. 후보로 출력하되 작업자가 원문 보고 결정.
